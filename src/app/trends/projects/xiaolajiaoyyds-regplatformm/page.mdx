# RegPlatform - Multi-Platform Account Registration System

RegPlatform is an automated batch account registration platform for OpenAI, Grok, Kiro, and Gemini services. It provides OAuth token acquisition, credit management, and multi-user administration through a distributed HuggingFace Space worker architecture.

## Key Features

- **Multi-platform support** - OpenAI, Grok, Kiro, Gemini automated registration
- **Distributed workers** - HuggingFace Space nodes with Cloudflare Worker routing  
- **Real-time monitoring** - WebSocket/SSE live task logging
- **Credit system** - User points management with redemption codes
- **Admin panel** - User management, system settings, worker node control
- **Elastic scaling** - Auto health checks and worker pool management
- **Proxy support** - Built-in proxy rotation for registration tasks

## Installation

<Callout type="warning">
Complex deployment requiring HuggingFace account keys, Git repository secrets, and Cloudflare configuration. Not recommended for beginners.
</Callout>

```bash
# Clone repository
git clone https://github.com/xiaolajiaoyyds/regplatformm.git
cd regplatformm

# Configure environment
cp .env.example .env
# Edit .env with database and API keys

# Local development
go mod tidy
go run cmd/server/main.go

# Frontend (separate terminal)
cd web && npm install && npm run dev

# Production deployment
docker-compose -f docker-compose.prod.yml up -d
```

## Basic Usage

```bash
# Access web interface
curl http://localhost:8000

# Create registration task via API
curl -X POST http://localhost:8000/api/tasks \
  -H "Authorization: Bearer $JWT_TOKEN" \
  -d '{"platform": "openai", "count": 10, "proxy": "http://proxy:8080"}'

# Start task
curl -X POST http://localhost:8000/api/tasks/123/start

# Monitor real-time logs
curl http://localhost:8000/ws/logs/123/stream
```

The first registered user automatically becomes admin. Configure HuggingFace tokens and worker spaces through the admin panel before running registration tasks.
