Contributing to the Agentic Playbook involves adhering to specific standards for guides. Each guide must be tested end-to-end, meaning you should run every command yourself before submitting it. It should be self-contained, allowing a reader to follow it from start to finish and achieve a working result. It's also important to be honest about any failures you encounter, so include a troubleshooting section that addresses real problems you faced. Lastly, make sure to timestamp your guide with a last tested date, so readers can determine if the information is current.

When you're ready to submit a guide, the process is straightforward. First, fork the repository. Then, create a new directory under the appropriate tool category in the source app folder. Write your guide as a page with the required frontmatter, and finally, open a pull request that includes a description of what you tested and the environment you used.

For structuring your guide, follow a specific template. Start with a title that clearly states what the guide will help the reader accomplish. Next, list any prerequisites, detailing what is needed before starting the guide. Follow this with the steps, which should be numbered and include actual commands and configurations. After that, provide a verification section that explains how to confirm the guide worked as intended. Finally, include a troubleshooting section that outlines common failures and their fixes.

Every guide page should begin with the GuideHeader component. This includes specifying the tool name, difficulty level, last tested date, and estimated reading time. Difficulty levels are categorized as beginner, which requires no prior experience with the tool; intermediate, which assumes basic familiarity; and advanced, which involves deep configuration, edge cases, and production hardening.

Currently, the playbook is accepting guides for several tools, including OpenClaw, which focuses on multi-agent gateway systems; Paperclip, for AI company orchestration; LangSmith Fleet, which deals with agent observability; Skills and Plugins, for extending agent capabilities; and Security, which covers production hardening across any tool. If you don’t see your tool listed, you can open an issue to propose a new tool category. If the tool is relevant to agentic workflows and you can provide a tested guide, it will be added.
