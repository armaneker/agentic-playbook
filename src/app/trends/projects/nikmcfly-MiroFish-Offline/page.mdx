# MiroFish-Offline

MiroFish-Offline is a fully local multi-agent simulation engine that generates hundreds of AI personas to predict public reactions to documents. This English fork of MiroFish replaces cloud dependencies with Neo4j + Ollama for complete offline operation.

## Key Features

- **Multi-agent simulation** - Creates hundreds of unique AI personas with distinct personalities and biases
- **Knowledge graph extraction** - Builds entity relationships from documents using local Neo4j
- **Social media simulation** - Agents post, argue, and shift opinions over time on simulated platforms  
- **Sentiment tracking** - Real-time analysis of opinion evolution and influence dynamics
- **Zero cloud dependencies** - Runs entirely on local hardware with Ollama LLMs
- **Full English UI** - Complete translation from original Chinese interface
- **Agent interaction** - Chat with simulated personas post-simulation

<Callout type="info">
Designed for PR crisis testing, policy impact analysis, and market sentiment prediction without requiring external APIs.
</Callout>

## Installation

### Docker (Recommended)

```bash
git clone https://github.com/nikmcfly/MiroFish-Offline.git
cd MiroFish-Offline
cp .env.example .env

# Start all services
docker compose up -d

# Pull required models
docker exec mirofish-ollama ollama pull qwen2.5:32b
docker exec mirofish-ollama ollama pull nomic-embed-text
```

### Manual Installation

```bash
# Start Neo4j
docker run -d --name neo4j \
  -p 7474:7474 -p 7687:7687 \
  -e NEO4J_AUTH=neo4j/mirofish \
  neo4j:5.15-community

# Start Ollama and pull models
ollama serve &
ollama pull qwen2.5:32b
ollama pull nomic-embed-text

# Backend
cd backend && pip install -r requirements.txt && python run.py

# Frontend  
cd frontend && npm install && npm run dev
```

## Basic Usage

1. **Upload document** - Feed the system a press release, policy draft, or financial report
2. **Graph extraction** - System builds knowledge graph of entities and relationships
3. **Agent generation** - Creates hundreds of personas with unique traits and biases
4. **Run simulation** - Agents interact on social platforms over simulated time periods
5. **Analyze results** - View sentiment evolution, influence dynamics, and generated content

<Callout type="warning">
Requires 16GB+ RAM and 10GB+ VRAM for optimal performance. Use smaller models (qwen2.5:14b) for resource-constrained setups.
</Callout>

Access the interface at `http://localhost:3000` after startup.

## Configuration

Key `.env` settings:

```bash
LLM_MODEL_NAME=qwen2.5:32b
NEO4J_URI=bolt://localhost:7687  
NEO4J_PASSWORD=mirofish
EMBEDDING_MODEL=nomic-embed-text
```
