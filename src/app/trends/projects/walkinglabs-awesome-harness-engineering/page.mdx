# Awesome Harness Engineering

Awesome Harness Engineering is a curated collection of resources for harness engineering—the practice of building reliable environments around AI agents. It focuses on making agents dependable in real workflows, especially for long-running coding and research tasks.

## Key Features

- **Foundational articles** from OpenAI, Anthropic, and other AI engineering teams
- **Context management** techniques for working memory and state handling
- **Safety patterns** including guardrails, sandboxing, and constraint design
- **Standardized specs** like AGENTS.md for repo-local agent instructions
- **Evaluation frameworks** for measuring agent reliability and performance
- **Reference implementations** of production harness architectures

## Installation

This is a curated list—no installation required. Simply browse the repository:

```bash
git clone https://github.com/walkinglabs/awesome-harness-engineering.git
cd awesome-harness-engineering
```

<Callout type="tip">
The repository includes a companion course at [learn-harness-engineering](https://github.com/walkinglabs/learn-harness-engineering) for hands-on practice.
</Callout>

## Basic Usage

The repository is organized by harness engineering domains:

```markdown
## Context, Memory & Working State
- Effective context engineering patterns
- KV-cache locality and tool masking
- Conversation memory management

## Constraints, Guardrails & Safe Autonomy  
- Sandboxing and permission models
- Prompt injection mitigation
- Quality assessment loops
```

Navigate to sections matching your current challenge—whether that's context window management, safety constraints, or evaluation design.
